# 贡献指南

> 项目还在很早期。**最有价值的贡献是"反对意见"**——告诉我们哪些假设站不住、哪些需求其实可以用现有工具解决。

## 怎么参与

### 1. 讨论需求(目前最需要的环节)

直接在 Issues 里开,建议加上前缀:

- **[需求]** 我希望工具支持识别 XXX 类别 / 场景
- **[反对]** 我觉得 REQUIREMENTS.md 第 X 条没必要
- **[现有]** 你们漏了 XXX 工具,它已经做到了 X 功能
- **[问题]** 关于 [REQUIREMENTS § 六](REQUIREMENTS.md) 的某个未决问题,我的意见是……

### 2. 写代码 / 做原型

技术栈尚未定稿(见 [REQUIREMENTS § 六](REQUIREMENTS.md))。

**若想动手写 PoC,先开 Issue 同步思路**——避免重复造轮子,也方便其他人提前看到你的方向。

可参考路径:

- 跑通某一类资产的"识别 + 体积统计"原型(比如先做"开发环境")
- 写一个 ai-disk-cleanup 风格的 CLI 验证 LLM 分类准确率
- 用 Tauri / Electron 做一个最小可用 GUI

### 3. 设计 / UX

我们急需一个"按业务类别浏览"的低保真原型。设计师特别欢迎。
现有产品(WizTree / DaisyDisk / TreeSize)的 UI 都不适合做参考——它们都是"按文件夹层级浏览"。

## 行为准则

直率讨论,对事不对人。
